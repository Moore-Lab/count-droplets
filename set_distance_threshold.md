# set_distance_threshold.py

Determines the optimal linking distance threshold for `track_droplets.py` by analyzing how the number of tracked droplets varies with the distance parameter. Fits a line + exponential model and identifies where the data departs from the linear baseline.

## Dependencies

```bash
pip install numpy matplotlib scipy
```

## Inputs

- **`droplets_vs_distance.npy`** - 1D NumPy array where index `i` is the number of tracked droplets when using linking distance `i` pixels. Generated by running `track_droplets.py` with `max_distance=None`.
- Alternatively, pass data directly via the `data` argument.

## Usage

### Standalone

```bash
python set_distance_threshold.py
```

Loads `droplets_vs_distance.npy` from the current directory, plots the analysis, and prints the suggested threshold.

### Called from track_droplets.py

When `track_droplets.py` is run with `max_distance=None`, it automatically scans thresholds 0-74, builds the distance array, and calls `plot_droplets_vs_distance()` to determine the best threshold.

### Programmatic Usage

```python
from set_distance_threshold import plot_droplets_vs_distance

best_threshold, n_tracked = plot_droplets_vs_distance(
    data=my_array,       # or filename="droplets_vs_distance.npy"
    fit_mode="joint",    # "joint" or "fixed_line"
    k_sigma=2.0,         # significance threshold
    two_sided=False,     # upward-only deviation
    show_plot=True
)
```

## Algorithm

### Model: Line + Exponential

```
y(x) = m*x + b + A * exp(-x / tau)
```

At large linking distances, true tracks plateau (linear baseline). At small distances, an exponential bump appears from fragmented short-range links.

### Fitting Modes

| Mode | Description |
|------|-------------|
| `"joint"` (default) | Fit all four parameters (m, b, A, tau) jointly via weighted least-squares |
| `"fixed_line"` | Fit m and b from the second half of the data (linear region), then fit only A and tau |

Weights are `1/sqrt(max(y, 1))` to handle Poisson-like count statistics.

### Threshold Selection

Scans from right to left along the x-axis and returns the first index where the data exceeds the linear baseline by at least `k_sigma` error bars:

```
deviation >= k_sigma * sqrt(max(y, 1))
```

- `two_sided=False` (default): only upward deviations trigger
- `two_sided=True`: absolute deviations trigger
- Fallback: if no crossing is found, uses the maximum first-difference (elbow heuristic)

## Plot Output

Two-panel figure:

1. **Top panel**: Data with error bars, linear baseline (dashed red), line + exponential fit (green), and threshold marker (black dashed vertical line).
2. **Bottom panel**: Normalized residuals with reduced chi^2.

## Key Parameters

| Parameter | Default | Description |
|-----------|---------|-------------|
| `fit_mode` | `"joint"` | `"joint"` or `"fixed_line"` |
| `k_sigma` | 2.0 | Number of error bars for threshold detection |
| `two_sided` | False | Whether to use absolute deviation |
| `show_plot` | True | Display the diagnostic plot |

## Returns

```python
(threshold_index, n_droplets_at_threshold)
```

- `threshold_index`: suggested optimal linking distance in pixels
- `n_droplets_at_threshold`: number of droplets tracked at that distance
